2. Clone & Build

Standalone build

# Clone with submodules
git clone --recurse-submodules https://github.com/naschpitz/ANN.git
cd ANN

# Create build directory
mkdir build && cd build

# Configure & build
cmake ..
make -j$(nproc)
The library builds as a static library (libANN.a). It explicitly instantiates templates for int, float, and double.

Running tests

# From the project root
./build/test_ann

3. Integrate into Your Project

Option A — Git submodule (recommended)

# Add ANN as a submodule
git submodule add https://github.com/naschpitz/ANN.git extern/ANN
git submodule update --init --recursive

Then in your CMakeLists.txt:

add_subdirectory(extern/ANN)
target_link_libraries(your_target PUBLIC ANN)

Option B — System install

cd build
sudo make install

4. Minimal Example

#include "ANN_Core.hpp"
#include "ANN_CoreConfig.hpp"
#include "ANN_Sample.hpp"

int main() {
    // Configure the network
    ANN::CoreConfig<float> config;
    config.modeType    = ANN::ModeType::TRAIN;
    config.deviceType  = ANN::DeviceType::CPU;
    config.layersConfig = makeLayersConfig({
        {3, ANN::ActvFuncType::NONE},
        {4, ANN::ActvFuncType::RELU},
        {1, ANN::ActvFuncType::SIGMOID}
    });
    config.trainingConfig.numEpochs    = 100;
    config.trainingConfig.batchSize    = 64;
    config.trainingConfig.learningRate = 0.01f;

    // Create and train
    auto core = ANN::Core<float>::makeCore(config);
    core->train(samples);

    // Extract trained parameters (for persistence by caller)
    const auto& params = core->getParameters();

    // Run a prediction
    ANN::Input<float> input = { 0.5, 0.3, 0.8 };
    ANN::Output<float> output = core->predict(input);

    return 0;
}
ANN is a pure computational library — it has no built-in file I/O. The calling application (e.g. NN-CLI) is responsible for loading configuration and saving trained models.
