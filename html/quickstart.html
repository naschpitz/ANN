2. Clone & Build

Standalone build

# Clone with submodules
git clone --recurse-submodules https://github.com/naschpitz/ANN.git
cd ANN

# Create build directory
mkdir build && cd build

# Configure & build
cmake ..
make -j$(nproc)
The library builds as a static library (libANN.a). It explicitly instantiates templates for int, float, and double.

Running tests

# From the project root
./build/test_ann

3. Integrate into Your Project

Option A — Git submodule (recommended)

# Add ANN as a submodule
git submodule add https://github.com/naschpitz/ANN.git extern/ANN
git submodule update --init --recursive

Then in your CMakeLists.txt:

add_subdirectory(extern/ANN)
target_link_libraries(your_target PUBLIC ANN)

Option B — System install

cd build
sudo make install

4. Minimal Example

#include "ANN_Core.hpp"
#include "ANN_Utils.hpp"

int main() {
    // Load a pre-trained model
    auto core = ANN::Utils<float>::load("model.json");

    // Run a prediction
    ANN::Input<float> input = { 0.5, 0.3, 0.8 };
    ANN::Output<float> output = core->predict(input);

    // output now contains the network's prediction
    return 0;
}

5. JSON Configuration Format

The JSON file encodes the entire network — architecture, hyperparameters, and learned weights:

{
  "device": "cpu",
  "mode": "train",
  "layersConfig": [
    { "numNeurons": 784, "actvFuncType": "relu" },
    { "numNeurons": 128, "actvFuncType": "relu" },
    { "numNeurons": 10,  "actvFuncType": "sigmoid" }
  ],
  "trainingConfig": {
    "numEpochs": 10,
    "learningRate": 0.01,
    "numThreads": 0,
    "progressReports": 1000
  }
}
When "numThreads" is 0, the library uses all available CPU cores. When "numGPUs" is 0, it uses all available GPUs.

After training, the "parameters" key is populated with learned "weights" and "biases", which allows the model to be loaded later for prediction or further training.
